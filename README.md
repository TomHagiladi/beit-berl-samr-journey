# מסע למידה: מודל SAMR - בית ברל

אפליקציית מסע למידה + מצגת מלווה למפגש 2 (הפרקטי) של צוות פיתוח AI בחינוך המיוחד,
מכללת בית ברל - "התערבות פדגוגית מובנית לשילוב בינה מלאכותית באמצעות מודל SAMR ווייב קודינג".

נבנה על תבנית `zpd-learning-journey` (מקצועות הבריאות, אונו): הוחלף התוכן, נשמר המנוע.

## מה יש כאן

| קובץ | תפקיד |
|------|--------|
| `index.html` | אפליקציית מסע הלמידה - הזנת שם + 4 תחנות |
| `explore.html` | ההתנסות הפותחת: "מציאת עניין בכל נושא?" (מבוסס על ההתנסות מקהילת הנוירופדגוגיה, בעיצוב המפגש) |
| `slides.html` | המצגת המלווה של המנחה (11 שקפים, שני QR וטיימרים מובנים) |
| `teacher.html` | לוח מנחה חי - הגשות המשתתפות בזמן אמת (לא מקושר מהאפליקציה) |
| `accessibility.html` | הצהרת נגישות (ת"י 5568 / WCAG 2.0 AA) |
| `tools/` | סקריפטים: יצירת QR, הזרקה למצגת, שליפה ופריסה של חוקי Firestore |

## מבנה המסע

1. פתיחה - הזנת שם.
2. היכרות עם המודל - פרומפט-אב שהופך את Gemini למורה אישית שמלמדת את מודל SAMR בדיאלוג; עיבוד אישי נשלח למנחה.
3. שיחה בזוגות - סיפורי ההשראה משיעורי הבית של מפגש 1 פוגשים את הסולם; תובנה משותפת נשלחת.
4. תוצר אישי - תיאור שיעור אמיתי, פרומפט שנבנה אוטומטית, ניתוח + הצעת שדרוג של מדרגה אחת; התוצר נשלח.
5. רפלקציית מראה - "המפגש הזה עצמו, איפה על הסולם?" - פותחת את הדיון המסכם.

## תשתית

- אתר סטטי על GitHub Pages (ללא build), ונילה HTML/CSS/JS.
- התקדמות המשתתפת נשמרת ב-localStorage (מפתח `samr-journey-v1`) - רענון לא מאבד כלום.
- הגשות טקסט נשמרות ב-Firestore - פרויקט `ono-health-gallery` (משותף לסדנאות),
  באוספים ייעודיים: `samr_reflections`, `samr_pairs`, `samr_products`, `samr_mirror`,
  `samr_progress`, `samr_admin`. חוקים: קריאה והוספה בלבד, עם אימות שדות.
- שליחות בשיטת "שגר ושכח" - האפליקציה ממשיכה לעבוד גם בלי רשת.

## התאמה לקהל אחר (תבנית לשימוש חוזר)

הסדנה בנויה כתבנית: מחליפים את **התוכן**, שומרים על **המבנה**.

מה מחליפים בדרך כלל:
1. פרומפט-האב של תחנה 1 (`textarea#p1`) + כרטיס המושג (`.concept`).
2. שאלות הדיון בזוגות (רשימת `q-list` ב-`index.html` + מערך `QUESTIONS` ב-`teacher.html` - חייבים להיות זהים).
3. תבנית הפרומפט של התוצר (`PB_HEAD`/`PB_TAIL` בראש הסקריפט).
4. מערך המדרגות `RUNGS` + דיאלוג הסולם (`rubric-dialog`) + השקף המקביל במצגת.
5. שמות האוספים ב-Firestore (חיפוש `samr_` בשני הקבצים) + עדכון חוקי האבטחה
   (`tools/fetch_rules.py` ואז `tools/deploy_rules.py` - תמיד לשלוף קודם! ה-NEW_RULES בסקריפט מתיישן).
6. זמני התחנות - בשלושה מקומות מתואמים: צ'יפים ב-`index.html`, `RECOMMENDED_MIN`, ו-`data-timer` במצגת.
7. יצירת QR חדש: `tools/make_qr.py` ואז `tools/inject_qr.py` (המצגת מכילה placeholder בשם `QR_SVG`).
8. `accessibility.html` - עדכון שם האתר והתאריך.
